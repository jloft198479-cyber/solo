## 教训
- 参考对象要选对（GitHub/Obsidian 的 callout 设计不够文艺，Notion 的软底设计才是对的）
- 用户要求参考成熟方案时，不要自己发明设计
- 依赖升级需谨慎（Vite 7→8 级联故障）
- 表格 CSS 应直接用 prosemirror-tables 官方默认值，不自造
- Tauri 前端 API 调用需在 capabilities JSON 中显式声明权限，否则静默失败不报错
- 图标设计必须对标品牌调性（Linear/Raycast 极简几何），不做粗糙占位

- 修复：移除 watcher 对 activeThemeId 的监听
- 提交：`8ecc480`
